Ambiguidade

Quando o mesmo código pode ter múltiplas interpretações - um problema crítico em compiladores

🔹 O que é

Uma gramática é ambígua quando uma mesma sentença pode ter mais de uma árvore sintática diferente.

Isso significa que a interpretação do código não é única, o que causa problemas sérios na análise sintática.

⚠️ Problema: O mesmo código pode produzir resultados diferentes dependendo da interpretação escolhida!

Exemplo Visual:

2 + 3 * 4
Interpretação 1:
(2 + 3) * 4 = 20
Interpretação 2:
2 + (3 * 4) = 14
Resultado: Duas respostas diferentes para a mesma expressão!

Problema da Ambiguidade

A mesma expressão gera múltiplas árvores sintáticas, causando incerteza na interpretação do código.

Consequência: O compilador não sabe qual interpretação usar, podendo gerar código incorreto ou até mesmo falhar na compilação.

🚫 Problema do "Dangling Else"

Um dos exemplos mais famosos de ambiguidade sintática. O else pode se associar a qualquer um dos ifs!

🔥 Código Ambíguo:

if (x > 0) if (y > 0) print('A') else print('B')

Pergunta: A qual if o else pertence? Com x = 1 e y = -1, o que acontece?

Interpretação 1: else associado ao if interno

O else se refere ao segundo if (if y > 0)

if (x > 0) { if (y > 0) print('A') else print('B') }
Valores de teste: x = 1, y = -1

🔍 Execução Passo a Passo

1x > 0? Verdadeiro → Entra no primeiro if👁️
2y > 0? Falso → else do if interno
3Executa: print('B')
4Resultado: B é impresso

📊 Comparação dos Resultados

Interpretação 1if interno (y > 0)BExecuta o else
Interpretação 2if externo (x > 0)NadaIgnora o else

⚠️ Problema: A mesma entrada produz resultados diferentes! Isso torna o comportamento do programa imprevisível e pode causar bugs sérios.

✅ Solução: Regras de Precedência

A maioria das linguagens resolve isso com a regra:"else sempre se associa ao if mais próximo"

// Interpretação padrão (Interpretação 1) if (x > 0) { if (y > 0) print('A') else print('B') // ← else associado ao if mais próximo }

Se você quiser a Interpretação 2, deve usar chaves explicitamente:

// Forçar Interpretação 2 com chaves if (x > 0) { if (y > 0) print('A') } else { print('B') }

🚫Problema Real

• Compiladores diferentes podem interpretar o mesmo código de formas distintas

• Bugs difíceis de detectar e reproduzir

• Comportamento inconsistente entre versões

• Vulnerabilidades de segurança em parsing

✅Soluções

• Definir precedência de operadores

• Estabelecer regras de associatividade

• Redesenhar a gramática para eliminar ambiguidade

• Usar parênteses para forçar interpretação
